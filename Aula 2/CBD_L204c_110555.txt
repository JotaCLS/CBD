Na implementação do redis tivemos que usar listas, sets, hashsets, etc para o armazenamento de dados, no mongo ao utilizar json senti que foi mais fácil adicionar o campo de quantidade, ou seja, se for preciso adicionar, alterar ou eliminar campos dos dados inseridos senti que o mongo é mais fácil nesse aspeto. Mas tendo em conta que neste sistema o intervalo de tempo em que os acessos à base de dados são relevantes, o redis é bastante rápido nas operações de leitura e escrita pois é executado diretamente na memória RAM enquanto o mongo em casos de muitos acessos simultâneos pode não ser tao rápido. Como a informação a ser guardado não é muito complexa e a velocidade de acesso à base de dados é um fator relevante eu optaria por usar o redis.
